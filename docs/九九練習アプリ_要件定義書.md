## 1. プロジェクト概要

### 1.1 目的

小学生が九九（掛け算）を効率的に練習・習得するためのWebアプリケーションを開発する。苦手な問題を重点的に出題する適応型学習アルゴリズムにより、個々の児童の習熟度に応じた学習体験を提供する。

### 1.2 対象ユーザー

小学2〜3年生（7〜9歳程度）を主な対象とする。ITリテラシーが十分でないことを前提に、直感的で簡単な操作性を重視する。

### 1.3 利用環境

ローカルPC上のブラウザで動作するオフラインWebアプリケーションとする。サーバー・インターネット接続は不要とし、単一のHTMLファイルをブラウザで開くだけで利用可能とする。

## 2. 技術スタック

| 項目 | 技術 | 選定理由 |
|------|------|----------|
| UIフレームワーク | React (JSX) | コンポーネント指向で画面遷移・状態管理がシンプル |

| 配布形態 | 単一 .html ファイル | ダブルクリックで起動、インストール不要 |

## 3. 機能要件

### 3.1 アカウント管理

#### 3.1.1 アカウント作成

- ユーザーは名前（ニックネーム）を入力するだけでアカウントを作成できる。
- パスワードは設定しない。
- アカウント情報は localStorage に保存する。
- アバターアイコンを選択可能とする（任意、デフォルトアイコンあり）。

#### 3.1.2 アカウント選択（ログイン）

- アプリ起動時、既存アカウントの一覧が表示される。
- ユーザーは自分の名前をタップ/クリックして選択するだけでログインできる。
- 「あたらしくつくる」ボタンから新規アカウント作成画面へ遷移する。

#### 3.1.3 アカウント削除

- ホーム画面の設定メニューからアカウント削除が可能。
- 削除時は確認ダイアログを表示する。
- 削除されたアカウントに関連するすべてのデータを削除する。対象は以下のとおり:
  - 成績データ（セッション履歴）
  - 問題ごとの重み
  - アカウント別設定（サウンド・エフェクトの ON/OFF・音量）
  - ログイン状態（削除対象が現在のログインユーザーの場合、ログアウト状態に戻す）
- 詳細な削除キー一覧は SPEC.md §2.5 `deleteAccount` の契約に従うこと。

### 3.2 画面構成

アプリケーションは以下の**5 画面**で構成する。結果サマリーは試験画面終了後に独立画面として表示する。

```
[ログイン画面] → [ホーム画面] → [試験画面] → [結果サマリー画面]
                      ↓                            │
               [成績表示画面]                        │
                      ↑                            │
                      └────────────────────────────┘
                              (ホームへ戻る)
```

#### 3.2.1 ログイン画面

| 項目 | 内容 |
|------|------|
| 目的 | アカウントの選択または新規作成 |
| 表示要素 | アプリタイトル、既存アカウント一覧、新規作成ボタン |
| 遷移先 | アカウント選択 → ホーム画面 / 新規作成 → アカウント作成フロー → ホーム画面 |

#### 3.2.2 ホーム画面

| 項目 | 内容 |
|------|------|
| 目的 | 試験の設定と開始、成績確認への導線 |
| 表示要素 | ユーザー名、試験設定（段の選択・問題数）、試験開始ボタン、成績ボタン、ログアウトボタン |
| 遷移先 | 試験開始 → 試験画面 / 成績 → 成績表示画面 / ログアウト → ログイン画面 |

#### 3.2.3 試験画面

| 項目 | 内容 |
|------|------|
| 目的 | 九九の問題に回答する |
| 表示要素 | 問題表示エリア、回答入力エリア（テンキー）、進捗表示、中断ボタン |
| 遷移先 | 全問回答完了 → 試験結果サマリー → ホーム画面 / 中断 → ホーム画面 |

#### 3.2.4 試験結果サマリー画面

| 項目 | 内容 |
|------|------|
| 目的 | 1 セッション終了後の成績と間違えた問題を確認する |
| 表示要素 | 正答数／出題数、正答率、所要時間、間違えた問題一覧、星評価、前回との比較（あれば） |
| 遷移先 | 「もういちど」→ 試験画面（同じ設定で再開）／「ホームへ」→ ホーム画面 |

#### 3.2.5 成績表示画面

| 項目 | 内容 |
|------|------|
| 目的 | 過去の成績・習熟度を確認する |
| 表示要素 | セッション一覧、段別正答率、苦手問題リスト、習熟度マップ |
| 遷移先 | 戻る → ホーム画面 |

### 3.3 試験機能

#### 3.3.1 出題範囲

- 1の段〜9の段を対象とする。
- ユーザーはホーム画面で出題する段を任意に選択できる（複数選択可）。
- 未選択時は全段（1〜9の段）から出題する。

#### 3.3.2 問題数

- 1セッションの問題数はユーザーが設定可能とする。
- デフォルト値: 27問
- 設定可能範囲: 10〜81問

#### 3.3.3 出題アルゴリズム

出題にはアダプティブ（適応型）アルゴリズムを採用する。

**重み付き抽出方式:**

1. 各問題（例: 7×8）に「重み（weight）」を持たせる。初期値は 1.0 とする。
2. 不正解の場合、その問題の重みを `+2.0` する。
3. 正解の場合、その問題の重みを `-0.3` する（下限 0.5）。
4. 重みはアカウントに紐付けて永続化し、セッションをまたいで蓄積する。
5. 出題時は、選択された段の問題プールから、重みに比例した確率で問題を抽選する。
6. 同一セッション内での連続重複出題は回避する（直前2問以内に出た問題は除外）。

#### 3.3.4 回答入力

- 画面に表示されるソフトウェアテンキー（0〜9）で回答を入力する。
- 物理キーボードの数字キー・テンキーからも入力可能とする。
- 入力後「こたえる」ボタン（またはEnterキー）で回答を確定する。
- 回答欄のクリア（全消去）ボタンを設ける。
- バックスペースキーおよび画面上の「けす」ボタンで1文字削除が可能。

#### 3.3.5 正誤フィードバック

- 正解時: 視覚的なエフェクト（○マーク、色の変化など）を表示し、短い遅延の後に次の問題へ自動遷移する。
- 不正解時: ×マークと正答を表示し、ユーザーが確認後に次の問題へ進むボタンを提示する。

#### 3.3.6 試験結果サマリー

セッション終了後に以下を表示する。

- 正答数 / 出題数
- 正答率（%）
- 所要時間
- 間違えた問題の一覧
- 前回からの改善ポイント（あれば）

### 3.4 成績管理

#### 3.4.1 保存データ

セッションごとに以下のデータを保存する。

| データ項目 | 型 | 説明 |
|-----------|-----|------|
| sessionId | string | セッション一意識別子 |
| accountId | string | アカウント識別子 |
| timestamp | number | セッション開始日時（UNIX時間） |
| selectedDans | number[] | 選択された段 |
| totalQuestions | number | 出題数 |
| correctCount | number | 正答数 |
| elapsedTime | number | 所要時間（秒） |
| details | object[] | 各問題の正誤詳細（問題・回答・正答・所要時間） |

#### 3.4.2 成績表示

- セッション履歴一覧（日時・正答率の時系列リスト）
- 段別の正答率一覧
- 九九81問の習熟度マップ（9×9のグリッドで色分け表示）
- 苦手問題ワースト5

## 4. 非機能要件

### 4.1 パフォーマンス

- 問題の遷移は200ms以内に完了すること。
- localStorage の読み書きは同期的に行い、体感上の遅延がないこと。

### 4.2 対応ブラウザ

- Google Chrome（最新版）
- Microsoft Edge（最新版）
- Firefox（最新版）
- Safari（最新版）

### 4.3 対応デバイス

- デスクトップPC（主要ターゲット）
- タブレット（レスポンシブ対応）

### 4.4 アクセシビリティ

- フォントサイズは十分に大きく設定する（本文16px以上、問題表示32px以上）。
- ボタンのタップ領域は最低48×48pxを確保する。
- 色覚多様性に配慮し、色だけに依存しない情報伝達を行う（○×マークの併用等）。
- ひらがな表記を基本とする（漢字使用時はふりがな付き）。

### 4.5 データ保全

- localStorage に保存するため、ブラウザのデータ消去により成績が失われる可能性がある。この点はユーザーに注意書きとして示す。
- **セッション履歴はアカウントあたり最新 100 件まで保持**する。それを超える古いセッションは自動的に破棄される（localStorage 5MB 上限への対策）。
- **ストレージ書き込み失敗時**（ブラウザ容量超過・プライベートモード等）は、子供向けに「ほぞん できなかったよ」と通知を出す。詳細は SPEC.md §2.4 ストレージ例外方針を参照。
- 将来的な拡張として、成績データのJSON エクスポート/インポート機能を検討する。

### 4.6 セキュリティ

- パスワード認証は行わない（要件に基づく設計判断）。
- 個人を特定する情報は保存しない（ニックネームのみ）。
- 外部通信は一切行わない。

## 5. データ設計

### 5.1 localStorage キー構成

```
kuku_accounts             : Account[]           // アカウント一覧
kuku_current_account_id   : string | null       // 最後に選択したアカウント ID（リロード自動復帰）
kuku_weights_{id}         : WeightMap           // アカウント別の問題重み
kuku_sessions_{id}        : SessionResult[]     // アカウント別のセッション履歴（最新 100 件まで）
kuku_settings_{id}        : Settings            // アカウント別の設定（サウンド・エフェクト・音量）
```

詳細な型定義は SPEC.md §2.2 を参照。

### 5.2 データ型定義

```typescript
// アカウント
interface Account {
  id: string;            // UUID
  name: string;          // ニックネーム
  avatarIndex: number;   // アバター番号
  createdAt: number;     // 作成日時
}

// 問題の重み（キー: "段-かける数" 例: "7-8"）
type WeightMap = Record<string, number>;

// セッション結果
interface SessionResult {
  sessionId: string;
  timestamp: number;
  selectedDans: number[];
  totalQuestions: number;
  correctCount: number;
  elapsedTime: number;
  details: QuestionDetail[];
}

// 問題詳細
interface QuestionDetail {
  dan: number;           // 段（例: 7）
  multiplier: number;    // かける数（例: 8）
  correctAnswer: number; // 正答（例: 56）
  userAnswer: number;    // ユーザーの回答
  isCorrect: boolean;    // 正誤
  responseTime: number;  // 回答時間（ミリ秒）
}
```

## 8. 将来拡張案（スコープ外）

以下は本バージョンのスコープ外とするが、将来的に検討する機能を記録する。

- **全網羅モード**: 重み付き抽選を無効化し、選択段の全問題を 1 回ずつ出題するモード（保護者の「全問練習」要望対応）
- **時間減衰機構**: 最終挑戦からの経過時間に応じて重みを徐々に引き上げる（得意問題の忘却対策）
- 成績データのエクスポート/インポート（JSON）
- 保護者向け管理画面
- 達成バッジ・トロフィー機能（モチベーション向上）
- タイムアタックモード
- 九九以外の計算（足し算・引き算・割り算）への拡張
- 複数端末間でのデータ同期
- PWA化によるオフラインインストール対応
- 音声読み上げ対応

## 9. 用語集

| 用語 | 説明 |
|------|------|
| 九九 | 1×1 から 9×9 までの掛け算の組み合わせ（81通り） |
| 段 | 掛け算の左側の数字（例: 7の段 = 7×1〜7×9） |
| セッション | 1回の試験（開始から終了まで）|
| 重み（weight） | 出題確率を制御する数値。大きいほど出題されやすい |
| 適応型アルゴリズム | ユーザーの正誤に応じて出題傾向を自動調整する仕組み |
